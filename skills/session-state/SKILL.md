# session-state スキル

## STATE.md の目的

`STATE.md` はセッション間で作業文脈を引き継ぐためのファイルです。

SessionStart hook が新規セッション開始時に自動的に `${プロジェクトルート}/.claude/STATE.md` を読み込み、Claude のコンテキストに注入します。これにより、「前回どこまで進んだか」「何が動いて何が残っているか」を毎回説明し直す手間がなくなります。

## STATE.md のテンプレート

```markdown
# プロジェクト状態

## 現在できていること
- （動作確認済みの機能・完了済み実装を箇条書き）

## 進行中のタスク
- （着手中だが未完了のもの。最優先を最上部に）

## 残タスク
- （未着手のもの。優先順に並べる）

## 既知の懸念・ブロッカー
- （バグ、未解決の設計問題、外部依存の不確実性など）

## 重要な決定事項へのリンク（docs/decisions/）
- （アーキテクチャ決定や方針変更の記録へのリンク）

## 最終更新
YYYY-MM-DD
```

## 書き方の原則

- **簡潔な箇条書き**で書く。散文は不要
- **絶対日付**を使う（「昨日」「先週」は次のセッションで意味を失う）
- **コードや git 履歴から自明な情報は書かない**（何を変更したかではなく、今何ができるかを書く）
- **次に着手すべきことを最上部**に置く（「進行中のタスク」が最優先情報）
- 細かすぎる実装メモより「どこまで到達したか」の状態を書く

## 配置場所

必ず**作業中プロジェクト**の `.claude/STATE.md` に保存すること。

```
/path/to/your-project/.claude/STATE.md  ← 正しい
~/.claude/STATE.md                       ← 間違い（グローバル設定ディレクトリ）
```

`.claude/` ディレクトリがなければ作成してよい。

## 更新方法

既存の `STATE.md` がある場合は**全置換せず差分更新**する。

1. 現在の `STATE.md` を読む
2. 完了したタスクを「現在できていること」に移動する
3. 新たに判明した懸念点・ブロッカーを追記する
4. 最終更新日を今日の日付に更新する
5. 「残タスク」と「進行中のタスク」を現状に合わせて整理する

## 発動タイミング

以下のタイミングで自発的に STATE.md の更新を提案・実行する:

- ユーザーが「状態を保存」「STATE更新」「作業を中断」「進捗まとめ」などと言ったとき
- 大きな機能の実装が完了したとき
- セッションが長くなってきた終盤
- 複数のサブタスクをまとめて完了したとき
